## 1. 정보설계 개요

### 목적

학습자가 목적에 따라 정보를 빠르게 찾고, 자연스럽게 다음 행동으로 이어지도록 정보의 구조·분류·레이블·흐름을 정의한다.

### 설계 기준

| 기준 | 내용 |
|------|------|
| 단순성 | 모바일 원핸드 환경. 탭 뎁스 최대 3단계 |
| 목적 우선 | 학습자가 "지금 무엇을 해야 하는지"를 항상 알 수 있어야 함 |
| 맥락 유지 | 어느 화면에서든 전체 여정에서 내가 어디에 있는지 인지 가능 |
| 역방향 이탈 최소화 | 완료 흐름 중 이탈을 방지하되, 강제하지 않음 |

### 사용자 멘탈 모델

학습자는 이 앱을 **"책 한 권을 함께 읽는 과정"**으로 인식한다.
- 챕터(모듈)를 순서대로 읽고
- 챕터 끝에 확인 테스트를 보고
- 다 읽은 뒤 얼마나 기억하는지 확인한다

정보 구조는 이 멘탈 모델을 따른다.

## 2. 앱 구조 (App Map)

```
팀플레이어 학습 도구
│
├── [온보딩] ─────────────────────── 최초 1회
│     ├── 서비스 소개 (3덕목 개념)
│     ├── 자기진단 유도
│     └── → 홈으로
│
├── [홈] ──────────────────────────── 바텀탭 #1
│     ├── 히어로 카드 (진행 현황 + "모델 알아보기" 탭)
│     │     └── → 모델 소개 (이상적인 팀 플레이어)
│     ├── 진행 현황 카드 (덕목별 N/N)
│     ├── 빠른 진입 CTA (이어서 학습하기)
│     └── → 모듈 목록 / 모듈 상세
│
├── [모델 소개] ─────────────────────── 슬라이드인 (홈 히어로에서 진입)
│     ├── 벤다이어그램 (3덕목 교집합 = 이상적인 팀 플레이어)
│     ├── 세 가지 덕목 (겸손/갈망/영리함)
│     ├── 7가지 유형 (겸손한 졸 / 불도저 / 사랑받는 연예인 / 돌발적인 사고뭉치
│     │     / 사랑스러운 게으름뱅이 / 노련한 정치가 / 이상적인 팀 플레이어)
│     └── 세 덕목이 막는 다섯 함정
│
├── [모듈 목록] ───────────────────── 바텀탭 #2
│     ├── 겸손 섹션 (M1~M3)
│     │     └── 모듈 카드 (완료 상태, 덕목 배지)
│     ├── 갈망 섹션 (M4~M5)
│     └── 영리함 섹션 (M6~M7)
│           └── → 모듈 상세
│
├── [모듈 상세] ───────────────────── 슬라이드인 (풀스크린)
│     ├── 헤더 (모듈명, 덕목 배지, 뒤로가기)
│     ├── 학습 목표 블록
│     ├── 탭: 학습
│     │     ├── 카드뉴스 캐러셀 (N장)
│     │     └── 체크리스트
│     ├── 탭: 연습 퀴즈
│     │     ├── 문항 캐러셀 (3문항)
│     │     ├── 즉시 정오답 피드백
│     │     └── 다시 풀기
│     └── 탭: 게이트
│           ├── 인트로 카드
│           ├── 문항 캐러셀 (5문항)
│           ├── 채점 버튼
│           └── 결과 바텀시트 (통과/미통과)
│
├── [자기진단] ────────────────────── 바텀탭 #3
│     ├── 안내 카드 (비판단적 설명)
│     ├── 설문 (18문항 / 덕목별 6문항 그룹)
│     ├── 제출 → 결과 뷰
│     │     ├── 덕목별 점수 바 차트
│     │     ├── 집중 포인트 카드
│     │     └── 다시 진단하기
│     └── (재진단 시) 사전-사후 비교 뷰
│
└── [나의 여정] ───────────────────── 바텀탭 #4
      ├── 타임라인 (모듈 완료 이력)
      ├── 착근기 항목 (Day+7 / Day+21)
      │     └── → 재인증 바텀시트
      │           ├── 3문항 (퀴즈 or 자기보고)
      │           ├── 실천 체크리스트
      │           └── 제출 → 완료 처리
      └── 성취 배지 목록 (예정)
```

## 3. 내비게이션 체계

### 3-1. 기본 내비게이션 구조

```
┌────────────────────────────────────┐
│          모듈 상세 (풀스크린)       │  ← 슬라이드인/아웃
│  뒤로가기 ←                        │
└────────────────────────────────────┘
        ↕ 진입/복귀
┌─────────────────────────────────────┐
│          콘텐츠 영역                 │
│  홈 | 모듈 목록 | 자기진단 | 여정   │
└────────────────────────────────────┘
│  홈  │  모듈  │  진단  │  여정  │  ← 바텀탭 (고정)
└──────┴────────┴────────┴────────┘
```

### 3-2. 바텀탭 구성

| 탭 순서 | 레이블 | 아이콘 설명 | 접근 조건 |
|---------|--------|------------|----------|
| 1 | 홈 | 집 모양 (house outline, 내부 계단 포함) | 항상 접근 가능 |
| 2 | 모듈 | 4개 사각형 격자 (2×2 grid) | 항상 접근 가능 |
| 3 | 진단 | 클립보드 + 체크마크 (문서에 ✓ 표시) | 항상 접근 가능 |
| 4 | 여정 | 타임라인 (3개 dot + 연결선 + 수평 항목선) | 항상 접근 가능 |

### 3-3. 화면 전환 방식

| 전환 유형 | 적용 화면 | 애니메이션 |
|-----------|-----------|-----------|
| 탭 전환 | 바텀탭 간 이동 | 페이드 또는 슬라이드 없음 (즉시) |
| 슬라이드인 | 모듈 목록 → 모듈 상세 | 오른쪽에서 슬라이드인 |
| 슬라이드아웃 | 모듈 상세 → 뒤로 | 오른쪽으로 슬라이드아웃 |
| 바텀시트 | 게이트 결과, 재인증, 진단 결과 | 아래에서 위로 슬라이드업 |
| 오버레이 | 게이트 채점 로딩 | 페이드인 (1.6초) |

### 3-4. 뒤로가기 & 이탈 처리

| 상황 | 처리 방식 |
|------|-----------|
| 모듈 상세에서 뒤로가기 | 모듈 목록으로 복귀, 진행 상태 유지 |
| 게이트 문항 중 뒤로가기 | 이탈 확인 없이 허용 (답안 초기화) |
| 자기진단 중 탭 이동 | 이탈 확인 없이 허용 (답안 임시 보존) |
| 재인증 바텀시트 닫기 | "여정으로 돌아가기" 버튼 또는 배경 탭 |

## 4. 화면 계층 & 전환 구조

### 4-1. 전체 화면 계층도

```
레벨 0 (앱 루트)
  └── 온보딩 (최초 1회)

레벨 1 (메인 탭)
  ├── 홈
  ├── 모듈 목록
  ├── 자기진단
  └── 나의 여정

레벨 2 (탭에서 진입하는 풀스크린)
  ├── 모듈 상세 (모듈 목록 or 홈 CTA에서 진입)
  └── 모델 소개 (홈 히어로 카드에서 진입)

레벨 3 (레벨 2 위에 올라오는 오버레이)
  ├── 게이트 결과 바텀시트
  └── 재인증 바텀시트 (나의 여정에서 진입)

레벨 3-대화형 (레벨 1 위에 올라오는 오버레이)
  └── 자기진단 결과 뷰 (자기진단 탭 내)
```

### 4-2. 모듈 상세 내부 구조

```
모듈 상세
│
├── 헤더 영역
│     ├── 뒤로가기 버튼
│     ├── 모듈 제목
│     └── 덕목 배지 (겸손/갈망/영리함)
│
├── 학습 목표 블록
│     └── 체크 아이콘 + 목표 텍스트 (2~3개)
│
└── 탭 영역
      ├── [학습 탭]
      │     ├── 카드뉴스 캐러셀
      │     │     ├── 슬라이드 N장 (스크롤 스냅)
      │     │     └── dot indicator
      │     └── 체크리스트
      │           ├── 항목 (탭으로 완료 처리)
      │           └── 완료 카운터 (N / N 완료)
      │
      ├── [연습 퀴즈 탭]
      │     ├── 문항 캐러셀 (3문항)
      │     │     ├── 문항 텍스트
      │     │     ├── 선택지 4개
      │     │     └── 즉시 피드백 (정답/오답 + 해설)
      │     └── 다시 풀기 버튼
      │
      └── [게이트 탭]
            ├── 인트로 카드 (안내 메시지)
            ├── 문항 캐러셀 (5문항)
            │     ├── 문항 텍스트
            │     ├── 선택지 4개
            │     └── 선택 상태 배지 (선택 전 / ✓ 선택됨)
            ├── 마지막 문항 힌트 (↓ 채점 안내)
            ├── 채점 버튼 영역 (구분선)
            └── 결과 바텀시트
                  ├── 통과: 점수 + 성취 메시지 + 다음 모듈 CTA
                  └── 미통과: 점수 + 단계적 멘토 안내 메시지(1/2/3회) + 재도전 유도 + 학습 탭으로 이동 CTA
```

### 5-2. 모듈별 콘텐츠 구성 기준

각 모듈은 아래 구성을 일관되게 따른다.

| 구성 요소 | 수량 | 형식 |
|----------|------|------|
| 카드뉴스 슬라이드 | 3~5장 | 제목 + 본문 텍스트 (또는 이미지) |
| 체크리스트 항목 | 2~4개 | 완료 가능한 행동 지향 문장 |
| 학습 목표 | 2~3개 | "~할 수 있다" 형식 |
| 연습 퀴즈 | 3문항 | 4지선다, 해설 포함 |
| 게이트 퀴즈 | 5문항 | 4지선다, 해설 없음 |

### 5-3. 자기진단 콘텐츠 구성

총 18문항. 패트릭 렌시오니의 자기참조형 문장("나는 ~") 형식이며, 덕목별 6문항씩 구성한다. (정본 05 퀴즈·진단 뱅크 기준)

| 덕목 | 문항 수 | 척도 | 점수 범위 |
|------|---------|------|----------|
| 겸손 (h) | 6문항 | 3점 척도 | 6~18점 |
| 갈망 (g) | 6문항 | 3점 척도 | 6~18점 |
| 영리함 (s) | 6문항 | 3점 척도 | 6~18점 |
| **합계** | **18문항** | | **18~54점** |

척도 레이블: `언제나(3)` / `때때로(2)` / `거의 아님(1)`

#### 결과 해석 (덕목별 6~18점)

| 점수 구간 | 해석 |
|----------|------|
| 16~18 | 강점 — 잘하고 있는 영역 |
| 13~15 | 점검 — 점검이 필요할 수 있는 영역 |
| 6~12 | 개발 — 더 키울 영역 |

이 진단은 점수를 매기는 평가가 아니라 **스스로를 비춰보는 성찰 도구**다. 결과는 가장 낮은 덕목 1개를 **개인 개발 초점**으로 제시하며, "가장 낮은 점수"가 아니라 "가장 크게 성장할 수 있는 영역"으로 안내한다.

## 6. 사용자 플로우

### 6-1. 핵심 플로우: 모듈 학습 → 게이트 통과

```
[홈 or 모듈 목록]
  학습 시작하기 →
        │
        ▼
[모듈 상세 > 학습 탭]
  카드뉴스 슬라이드 읽기 (1→N장)
  체크리스트 항목 완료
        │
        ▼
[모듈 상세 > 연습 퀴즈 탭]
  3문항 풀기 → 즉시 정오답 피드백
  (다시 풀기 가능)
        │
        ▼
[모듈 상세 > 게이트 탭]
  인트로 카드 확인
  5문항 선택
  채점하기 →
```

## 7. 정보 그룹화 원칙

### 7-1. 모듈 목록 그룹화

모듈은 **덕목 단위**로 그룹화한다. 총 3개 섹션.

```
겸손 (Humility) ── 황금색 계열
  M1 겸손의 의미와 중요성
  M2 겸손한 사람의 특징
  M3 겸손 실천하기

갈망 (Hunger) ── 파란색 계열
  M4 내적 갈망의 힘
  M5 자기동기부여

영리함 (Smarts) ── 초록색 계열
  M6 대인관계 영리함
  M7 팀 내 소통 영리함
```

그룹화 근거:
- 3덕목은 학습자가 이미 온보딩에서 인지한 개념 단위
- 덕목별 자기진단 점수와 연동해 학습 추천 맥락 연결 가능
- 색상 시스템(겸손=황금, 갈망=파랑, 영리함=초록)이 모든 화면에서 일관 적용

### 7-2. 자기진단 문항 그룹화

18문항을 덕목별 6문항씩 3개 섹션으로 표시.

그룹화 근거:
- 한 번에 18문항 나열 시 인지 부하 과다
- 덕목별 완료 시 소그룹 완료감 발생 (Zeigarnik 역이용)
- 결과 해석과 문항 그룹이 자연스럽게 연결

### 7-3. 타임라인 그룹화

나의 여정 타임라인은 **시간 순서** 기반으로 그룹화하되, 학습 단계별 소그룹을 가진다.

```
학습 전
  └── 자기진단

학습 중 (모듈별)
  ├── M1 완료 — 날짜
  ├── M2 완료 — 날짜
  └── ...

학습 완료
  └── 캡스톤 인증 — 날짜

착근기
  ├── Day+7 재인증 — 날짜
  └── Day+21 재인증 — 날짜
```

### 8-4. 덕목 레이블 체계

| 덕목 | 한글 레이블 | 영문 레이블 | 약어 | 색상 |
|------|-----------|-----------|------|------|

## 9. 상태 관리 & 조건부 노출

### 9-1. 모듈 상태 정의

| 상태 | 정의 | 시각 처리 |
|------|------|-----------|
| `locked` | 이전 모듈 미완료 (순차 개방 시) | 흐린 스타일, 탭 불가 |
| `available` | 진입 가능, 아직 시작 안 함 | 일반 스타일 |
| `in_progress` | 학습 시작 후 게이트 미통과 | "학습 중" 배지 |
| `completed` | 게이트 통과 | ✓ 완료 dot, 날짜 |

### 9-2. 조건부 노출 규칙

| UI 요소 | 노출 조건 | 미충족 시 처리 |
|---------|----------|--------------|
| 게이트 탭 | 항상 접근 가능 | (연습 퀴즈 먼저 권장하는 UX) |
| 게이트 채점 버튼 | 상시 활성 | 제출 시 미선택 문항 안내 + 해당 카드로 자동 스크롤 |
| Day+7 재인증 버튼 | 인증일 + 7일 경과 | 흐린 스타일 + "7일 후 활성화" |
| Day+21 재인증 버튼 | 인증일 + 21일 경과 | 흐린 스타일 + "21일 후 활성화" |
| 자기진단 제출 버튼 | 상시 활성 | 제출 시 "모든 항목(N/18)을 선택해주세요" 안내 |
| 다음 모듈 CTA | 게이트 통과 결과 시트에서 | 통과 시에만 노출 |
| 재도전 CTA | 게이트 미통과 결과 시트에서 | 미통과 시에만 노출 |
| 멘토 도움 안내 메시지 | 게이트 미통과 결과 시트에서 | 미통과 횟수(`gateFails`)에 따라 단계적 노출 |

### 9-4. 게이트 미통과 추적 & 멘토 알림

모듈별 게이트 미통과 횟수를 `gateFails`로 영구 누적 추적한다. 미통과 시 결과 시트에 횟수별로 다른 메시지를 노출하며, **3회 미통과 시 멘토에게 도움 메시지가 이메일로 자동 발송**된다. (알림 채널은 이메일 단일이며 카카오워크는 사용하지 않는다.)

| 누적 미통과 | 결과 시트 메시지 | 멘토 알림 |
|------------|----------------|----------|
| 1회 | "괜찮습니다. 한번 더 공부하고 다시 도전해봐요." (3회 실패 시 멘토 자동 발송 예고) | 없음 |
| 2회 | "학습에 어려움이 있으신가요? 자료를 한번 더 읽어보고 재도전 해봐요." (3회 실패 시 멘토 자동 발송 예고) | 없음 |
| 3회 | "멘토와 함께 이 난관을 헤쳐나가 보아요. 멘토에게 도움의 메시지가 자동 발송됩니다." | 멘토에게 도움 메시지 이메일 자동 발송 |

### 9-3. 진행 상태 저장 범위

| 데이터 | 저장 범위 | 초기화 시점 |
|--------|----------|------------|
| 모듈 완료 여부 | 영구 저장 | 없음 |
| 게이트 답안 | 채점 후 초기화 | 채점 시 또는 탭 이탈 시 |
| 게이트 미통과 횟수 (`gateFails`) | 모듈별 영구 누적 | 없음 (데이터 초기화 시만) |
| 연습 퀴즈 답안 | 세션 내 유지 | "다시 풀기" 탭 시 |
| 자기진단 답안 | 세션 내 유지 | "다시 진단하기" 탭 시 |
| 체크리스트 상태 | 모듈별 영구 저장 | 없음 |
| 재인증 완료 여부 | 영구 저장 | 없음 |

```
멘토 대시보드
```
