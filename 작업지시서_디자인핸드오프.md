## 3. 완성된 인터랙션 목록

### 모듈 상세 (`s-module`)
- **슬라이드 전환**: 모듈 목록 → 상세로 오른쪽에서 슬라이드인 (`.slide-in`/`.slide-out` keyframe)
- **목표 확인 블록**: 덕목 색상 체크 아이콘 + 학습 목표 2~3개 렌더 (`openModule()` 내)
- **탭 전환**: 학습 / 연습 퀴즈 / 게이트 (`switchTab()`)
- **학습 탭**: 카드뉴스 캐러셀 (`cn-track`, scroll-snap), 체크리스트 (`toggleCheck()`), dot indicator
- **연습 퀴즈 탭**: 캐러셀 형식 3문항, 정오답 즉시 표시, 해설 노출, "다시 풀기" (`retryPQ()`)
- **게이트 탭**: 인트로 카드 + 5문항 캐러셀, 전체 선택 후 채점 (`submitGate()`), 로딩 1.6초 → 결과 바텀시트

### 자기진단 (`s-diag`)
- `renderSurvey()` — 18문항 (덕목별 6개) 동적 렌더
- `pickSurvey(key, val)` — 언제나(3)/때때로(2)/거의 안 함(1) 선택, 색상 구분
- `submitDiag()` — 덕목별 점수 바 차트 + 비판단적 피드백 + 집중 포인트
- `resetDiag()` — 초기화 후 재진단

### 착근기 재인증 (`s-timeline`)
- `openRecert(day)` — Day+7/Day+21 바텀시트 열기, 3문항 + 실천 체크리스트
- `submitRecert(day)` — 완료 처리 + 타임라인 항목 활성화
- `closeRecertSheet()` → `goTo('timeline')` (여정으로 돌아가기)

## 4. 데이터 구조

### MODS 배열 (모듈 7개)
```js
{ id, n, title, virtue, vLabel, emoji, desc, objectives[] }
// virtue: 'h' | 'g' | 's'
// objectives: 학습 목표 2~3개
```

### SURVEY_DATA (자기진단 설문)
```js
{ h: { label, color, dark, bg, modRange, items[6] },
  g: { ... },
  s: { ... } }
// 척도: 언제나(3점), 때때로(2점), 거의 안 함(1점), Max 18점/덕목
```

### RECERT_DATA (재인증)
```js
{ 7: { title, subtitle, questions[3], practiceItems[3] },
  21: { ... } }
// questions[].ans === null이면 자기보고 문항 (정오답 없음)
```

## 6. 미완성 / 디자인 개선 필요 항목

### 🔴 High — 기능 미완
| 항목 | 현황 | 필요 작업 |
|------|------|-----------|
| **M2~M7 학습 콘텐츠** | M1만 실제 콘텐츠 작성됨. M2~M7은 desc만 있음 | 각 모듈 카드뉴스 슬라이드 3~4장 콘텐츠 작성 |
| **게이트 문항 모듈별 교체** | 현재 GQ[]가 M1 겸손 기준 5문항으로 고정 | `openModule()` 시 해당 덕목 GQ 교체 로직 추가 |
| **연습 퀴즈 모듈별 교체** | PQ[]도 M1 고정 | 위와 동일 |
| **홈 성장 현황** | 진행률 숫자가 모두 `0`으로 고정 | 모듈 완료 상태 저장 → localStorage 또는 JS state |

### 🟡 Medium — 시각 완성도
| 항목 | 현황 | 필요 작업 |
|------|------|-----------|
| **홈 화면 레이아웃** | 인사 + 블롭 Venn + 모듈 진행 카드가 있지만 여백·리듬 미정제 | 섹션 간격, 카드 정렬 재검토 |
| **다크 모드** | CSS 변수 준비됐지만 다크 테마 미적용 | `prefers-color-scheme: dark` 분기 추가 |
| **게이트 결과 시트** | 통과/미통과 분기 UI는 있지만 스타일 거침 | 통과 시 confetti 또는 pulse 애니메이션 |
| **자기진단 결과 바 차트** | 기능 작동. 폰트 크기·간격 개선 여지 | 결과 카드 레이아웃 정제 |
| **타임라인 완료 처리 시각화** | 재인증 완료 시 dot만 채워짐 | 완료 항목에 녹색 체크 배지 + 날짜 표시 |

### 🟢 Low — 추가 기능
| 항목 | 설명 |
|------|------|
| 온보딩 플로우 | 첫 진입 시 자기진단으로 유도하는 시작 화면 |
| 모듈 완료 인증 애니메이션 | 게이트 통과 시 전체 화면 축하 모먼트 |
