# O365 툴 vs 클로드코드 웹 앱 비교
> IT 팀 환경 세팅 지원 있음 조건

| 구분 | O365 툴 (Forms · Lists · Power Automate · Power BI) | 클로드코드 웹 앱 (기획자 + AI) | 판단 |
|------|------------------------------------------------------|-------------------------------|------|
| 초기 진입 장벽 | Forms·Lists 는 쉬움, Power Automate·Power BI 는 부담 큼 | IT 팀 환경 세팅 지원으로 초기 부담 해소. 이후 기획자가 AI와 함께 유지 | 비슷 |
| 보안 / 권한 | Microsoft 365 통합, 회사 정책과 잘 맞음 | IT 팀이 Azure AD 연동·사내 정책 정합 지원. 약점 해소 가능 | 비슷 |
| 기획자 역량과 맞음 | 자동화 흐름·계산식 중심, 기획 역량과 동떨어짐 | 기획·UX 중심, 코드는 AI 가 담당 | 웹 앱 ✓ |
| 사용자 경험 (UX) | "관리자용 데이터 입력창"처럼 보임, 학습 감각과 맞지 않음 | 학습 앱 형태로 설계 가능, 몰입감·감각 높음 | 웹 앱 ✓ |
| 학습 몰입도 | Forms 퀴즈 UI = 설문지 외관. 스토리·비네트 진입 의도를 살리기 어려움 | 스토리 진입, 7유형 비네트, 롤플레이 카드 등 콘텐츠 설계 의도를 UI로 구현 가능 | 웹 앱 ✓ |
| 퀴즈 / 학습 경험 | 단순 설문 경험, 해설·학습 경로 설계 제한 | 학습 경로, 즉시 해설, 리더보드 등 풍부하게 설계 가능 | 웹 앱 ✓ |
| 콘텐츠 업데이트 용이성 | 문항 하나 수정도 Forms·Automate 흐름 점검 필요. IT 빌더 의존 지속 | 기획자가 AI와 직접 수정, 흐름 깨짐 없이 빠른 반영 가능 | 웹 앱 ✓ |
| 운영 화면 | Lists = 데이터 테이블 외관, 비기술자 부담 큼 | 기획자가 비기술자용 운영 화면 직접 설계 가능 | 웹 앱 ✓ |
| 운영 대시보드 | Power BI 의존, 학습 곡선 큼 | 기획자가 지표 정의 + AI 구현, 시각적으로 이해 쉬움 | 웹 앱 ✓ |
| 운영 부담 | 기획자 집중, 미술 전공자는 "운영 불가" 느낌 | 기획자 + AI 분산, 미술 전공자도 운영 가능한 UI 설계 가능 | 웹 앱 ✓ |
| 유지보수 부담 | 흐름·대시보드 복잡해지면 누가 만들었는지 파악 어려움 | 코드 구조 명확하면 상대적으로 낮음. 단, 기획자 이탈 시 리스크 있음 | 비슷 |
| 지속 가능성 | 시범 운영 가능, 6개월 이상 조직 전체 운영은 어려움 | 기획자 강점 살려 6개월 이상 조직 전체 운영 가능 | 웹 앱 ✓ |
| 확장성 | Microsoft 생태계 내에서만 강력, 외부 연결 제한 | 외부 시스템·API 연결 자유롭고, 기능 추가 유연 | 웹 앱 ✓ |
| 초기 비용 | 라이선스 비용 낮음, 구현 비용 적음 | 개발 비용 더 나올 수 있음. 단, 한 번 만들면 확장성 높음 | O365 ✓ |
| 추천 대상 | 시범 운영, 1–2명 참여, 짧은 기간 사용 목적 | 여러 명이 함께 배우고, 6개월 이상 조직 전체 운영, IT 지원 확보된 경우 | 웹 앱 ✓ |
