- **벤다이어그램**: 세 원(겸손/갈망/영리함)이 순차 등장(`vennPop`, delay 0/.22/.44s) → 라벨 페이드인 → 가운데 "이상적인 팀 플레이어" 문구가 팝업 후 반짝임 루프(`idealPop` + `idealShimmer`)

## 3. 완성된 인터랙션 목록

### 모듈 상세 (`s-module`)
- **슬라이드 전환**: 모듈 목록 → 상세로 오른쪽에서 슬라이드인 (`.slide-in`/`.slide-out` keyframe)
- **목표 확인 블록**: 덕목 색상 체크 아이콘 + 학습 목표 2~3개 렌더 (`openModule()` 내)
- **탭 전환**: 학습 / 연습 퀴즈 / 게이트 (`switchTab()`)
- **학습 탭**: 카드뉴스 캐러셀 (`cn-track`, scroll-snap), 체크리스트 (`toggleCheck()`), dot indicator
- **연습 퀴즈 탭**: 캐러셀 형식 3문항, 정오답 즉시 표시, 해설 노출, "다시 풀기" (`retryPQ()`)
- **게이트 탭**: 인트로 카드 + 5문항 캐러셀, 전체 선택 후 채점 (`submitGate()`), 로딩 → 결과 바텀시트
  - **응시 UX(버튼 활성/비활성)**: 하단 `게이트 채점하기`는 전 문항 선택 전까지 `disabled`. 각 문항의 `다음 문항`도 해당 문항 선택 전까지 `disabled`이며, 선택(`pickGate()`) 직후 활성화됨. 전 문항 응답이 끝나면 채점 버튼이 활성화됨
  - **통과 시**: 모듈 완료 처리(`completeModule()`) + 폭죽 애니메이션(`launchConfetti()`)
  - **미통과 시 멘토 알림(단계 메시지)**: 누적 실패 횟수(`state.gateFails[cur]`)에 따라 메시지가 단계적으로 강해짐
    - 1회: "괜찮습니다. 한번 더 공부하고 다시 도전해봐요." (+ "3회 실패 시 멘토에게 도움 메시지가 자동 발송될 예정입니다.")
    - 2회: "학습에 어려움이 있으신가요? 자료를 한번 더 읽어보고 재도전 해봐요." (+ 동일 안내)
    - 3회: "멘토와 함께 이 난관을 헤쳐나가 보아요. 멘토에게 도움의 메시지가 자동 발송됩니다." → **멘토 이메일 자동 발송**. 알림 채널은 **이메일**

### 자기진단 (`s-diag`)
- `renderSurvey()` — 공식 18문항 (덕목별 6개) 동적 렌더. 렌시오니 자기참조형 문장("나는 ~한다") 사용, 출처 `05 퀴즈·진단 뱅크(정본)`
- `pickSv(key, val, vk)` — 척도 **언제나(3) / 때때로(2) / 거의 아님(1)** 선택, 덕목 색상 구분, 진행 카운터 갱신
- `submitDiag()` → `renderDiagResult()` — 덕목별 점수 바 차트(6~18점) + 개인 개발 초점 + 성찰 도구 프레이밍
- **결과 해석 기준(덕목별 6~18점)**: 16~18 강점 / 13~15 점검 필요 / 6~12 개발 영역
- **프레이밍**: 점수를 매기는 평가가 아니라 "스스로를 비춰보는 성찰 도구"(🪞). 가장 낮은 점수는 "가장 크게 성장할 수 있는 영역"으로 긍정 제시
- 재진단: 결과 화면의 `↺ 다시 진단하기`가 `renderSurvey()` 재호출

### 착근기 재인증 (`s-timeline`)
- `openRecert(day)` — Day+7/Day+21 바텀시트 열기, 3문항 + 실천 체크리스트
- `submitRecert(day)` — 완료 처리 + 타임라인 항목 활성화
- `closeRecertSheet()` → `goTo('timeline')` (여정으로 돌아가기)

> 전반적으로 `@media (prefers-reduced-motion: reduce)` 분기를 갖춰, 모션 최소화 설정 시 애니메이션을 끄거나 정적 상태로 고정합니다.

| 위치 | 모션 / 이펙트 | 비고 |
|------|---------------|------|
| 홈 히어로 | 유기적 **orb 루프** (3개 블롭이 서로 다른 주기로 드리프트, blur) | `drift1~3`, reduced-motion 시 정지 |
| 채워진 CTA | **광택 스윕** + **그림자 호흡**(`cta-breathe`) | `.btn-ink/.btn-h/.btn-g/.btn-s` |
| `다음 문항` 버튼 | 화살표 **넛지**(좌우 살짝 이동, `nudge`) | 버튼 `disabled` 시 넛지 정지 |
| 비활성 버튼 | **흐림/디밍** 처리 | 선택 전 게이트 다음/채점 버튼 |
| 학습 캐러셀 카드 | 테두리 **빛반사**(빛이 가장자리를 도는 `cardsheen`) | `.cn-card::before`, reduced-motion 시 정지 |
| 게이트 통과 | **폭죽**(confetti) | `launchConfetti()` |
| 모델 소개 벤다이어그램 | **순차 등장**(`vennPop`) + 이상 문구 **반짝임**(`idealShimmer`) | delay 0/.22/.44s |
| 모델 소개 섹션 | **스크롤 리빌**(`.reveal`, IntersectionObserver) | reduced-motion 시 즉시 표시 |
| 모듈 상세 | **아이콘/칩 영역 대비**(`md-hero` 등 시각 강조) | 덕목 색상 기반 |

## 4. 데이터 구조

### MODS 배열 (모듈 7개)
```js
{ id, n, title, virtue, vLabel, emoji, desc, objectives[] }
// virtue: 'h' | 'g' | 's'
// objectives: 학습 목표 2~3개
```

### SURVEY / SCALE (자기진단 설문)
```js
SURVEY = { h:[...6], g:[...6], s:[...6] } // 공식 18문항, 렌시오니 자기참조형("나는 ~한다")
SCALE  = [{v:3,t:'언제나'},{v:2,t:'때때로'},{v:1,t:'거의 아님'}]
// 덕목별 6문항 · 6~18점, 해석: 16~18 강점 / 13~15 점검 / 6~12 개발
```

### RECERT_DATA (재인증)
```js
{ 7: { title, subtitle, questions[3], practiceItems[3] },
  21: { ... } }
// questions[].ans === null이면 자기보고 문항 (정오답 없음)
```

| 항목 | 현황 | 필요 작업 |
