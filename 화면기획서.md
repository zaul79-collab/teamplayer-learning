## 1. 프로젝트 개요

### 배경

케이옥션 IT팀 AI-native 전환 과정에서 조직 역량 내재화 목적으로 Patrick Lencioni의 *The Ideal Team Player* 기반 팀플레이어 학습 시스템을 구축한다.

### 학습 대상 — 3가지 덕목

| 덕목 | 영문 | 핵심 정의 | 모듈 |
|------|------|-----------|------|
| 겸손 | Humility | 팀의 성공을 개인의 성공보다 중요하게 여기는 태도 | M1 · M2 · M3 |
| 갈망 | Hunger | 외부 동기 없이 스스로 더 잘하려는 내적 의지 | M4 · M5 |
| 영리함 | Smarts | IQ가 아닌 사람의 감정과 상황을 읽는 능력 | M6 · M7 |

### 학습 여정 개요

```
자기진단 → 모듈 학습 (M1~M7) → 게이트 인증 → 착근기 (Day+7 / Day+21)
```

- 인증은 완성이 아닌 출발선
- 반복은 시스템에, 판단과 코칭은 사람에게

### 플랫폼

모바일 웹 앱 · 430px 기준 · 단일 HTML 파일 (prototype.html)

## 2. 화면 구조 및 플로우

### 화면 목록

| ID | 화면명 | 진입 방법 |
|----|--------|-----------|
| `s-home` | 홈 | 하단 네비 "홈" |
| `s-about` | 모델 소개 (이상적인 팀 플레이어) | 홈 히어로 카드 탭 |
| `s-modules` | 모듈 목록 | 하단 네비 "모듈" |
| `s-module` | 모듈 상세 | 모듈 카드 탭 |
| `s-diag` | 자기진단 | 하단 네비 "진단" |
| `s-timeline` | 나의 여정 | 하단 네비 "여정" |

### 주요 사용자 플로우

```
[앱 첫 진입]
    │
    ▼
[홈 화면] ──── 히어로 카드 탭 ───→ [모델 소개] (슬라이드인, 뒤로가기로 복귀)
    │
    ▼
[자기진단] → 18문항 응답 → 결과 확인 → [모듈 학습 시작하기]
                                            │
                                            ▼
                                    [모듈 목록]
                                            │
                                            ▼
                                    [모듈 상세]
                                     ├─ 학습 탭 (카드뉴스 → 체크리스트)
                                     ├─ 연습 퀴즈 탭 (3문항 캐러셀)
                                     └─ 게이트 탭 (5문항 → 채점 → 바텀시트)
                                            │ 게이트 통과
                                            ▼
                                    [나의 여정] 타임라인 기록
                                            │ Day+7 활성화
                                            ▼
                                    [재인증 바텀시트] × 2회 (Day+7, Day+21)
```

### 화면 전환 방식

| 전환 | 방법 |
|------|------|
| 탭 간 이동 | 즉시 전환 (애니메이션 없음) |
| 홈 → 모델 소개 | 우측에서 슬라이드인 (300ms), 뒤로가기로 슬라이드아웃 |
| 모듈 목록 → 상세 | 오른쪽에서 슬라이드인 (280ms) |
| 모듈 상세 → 목록 | 오른쪽으로 슬라이드아웃 (250ms) |
| 게이트 결과, 재인증 | 바텀시트 슬라이드업 (380ms) |

## 3. 홈 화면

**화면 ID** `s-home`

### 구성 요소

| 영역 | 내용 | 비고 |
|------|------|------|
| 인사 영역 | 사용자 이름 + 부제 텍스트 | 상단 고정 |
| 히어로 카드 | 진행 현황(N/7 모듈) + 유기적 orb 블롭 + "모델 알아보기 →" 칩 | 탭 시 모델 소개(`s-about`) 슬라이드인 |
| 벤 다이어그램 | 3덕목 블롭 애니메이션 | CSS border-radius keyframe, 겹침 영역 = 이상적 팀플레이어 |
| 진행 현황 카드 | 덕목별 완료 모듈 수 | 겸손 0/3 · 갈망 0/2 · 영리함 0/2 |
| 빠른 진입 CTA | 현재 진행 모듈 바로가기 | 상태에 따라 동적 표시 |

### 벤 다이어그램 명세

- 세 블롭이 유기적으로 형태 변화하며 서로 겹침
- 각 블롭: 덕목 색상 `rgba()` 24% 투명도 + 레이블 텍스트
- 겹침 중앙: "팀플레이어" 텍스트

## 4. 모델 소개 화면 — 이상적인 팀 플레이어

**화면 ID** `s-about`

### 진입 / 이탈

- 진입: 홈 히어로 카드 탭(우상단 "모델 알아보기 →" 칩) → 우측에서 슬라이드인 패널 (`#s-about.in`)
- 이탈: 헤더 뒤로가기(←)로 슬라이드아웃 → 홈 복귀
- 헤더: 뒤로가기 + "이상적인 팀 플레이어" 타이틀 (sticky)

### 구성 (위→아래 세로 스크롤, 섹션 순차 노출)

| 순번 | 영역 | 내용 |
|------|------|------|
| ① | 인트로 | The Ideal Team Player — 패트릭 렌시오니 모델 소개. "**'이상적'은 '완벽'이 아니다**" — 세 덕목을 충분히 갖춰 함께 일하기 좋은 사람 |
| ② | 벤 다이어그램 | 겸손·갈망·영리함 3원(SVG), 세 원 교집합 = "이상적인 팀 플레이어". 원이 순차 등장(pop) + 중앙 문구 반짝임(shimmer) 루프 |
| ③ | 세 가지 덕목 카드 | 덕목별 카드 3개 — 겸손(신뢰의 토대) · 갈망(내적 의지, 변화 가장 어려움) · 영리함(대인관계 분별력, IQ 아님) |
| ④ | 7가지 유형 | 덕목 조합별 7유형 (아래 표). 유형명은 자기인식·코칭용 — 실재 동료에게 꼬리표로 붙이지 않음 (경고 박스) |
| ⑤ | 세 덕목이 막는 다섯 함정 | 렌시오니 '팀의 다섯 가지 함정'(1→5) + 세 덕목과의 연결 |
| ⑥ | CTA | "학습 모듈 보러가기 →" (소개 닫고 모듈 목록 이동) |

### 7가지 유형 목록

| 유형 | 덕목 조합 (겸·갈·영) | 특징 | 위험도 |
|------|------|------|--------|
| 겸손한 졸 | 겸 | 예의 바르나 성사 욕구·관계 구축 부족, 풍파는 없음 | 낮음 |
| 불도저 | 갈 | 의지는 확고하나 타인의 영향을 무시, 팀을 빠르게 파괴 | 중간 |
| 사랑받는 연예인 | 영 | 재미·호감을 주나 기여는 미미, 곧 호감을 잃음 | 중간 |
| 돌발적인 사고뭉치 | 겸·갈 | 진심으로 기여하나 대인 영리함 부족 (악의 없음) | 낮음 |
| 사랑스러운 게으름뱅이 | 겸·영 | 동료를 배려하나 딱 요구받은 것만, 관리 필요 | 중간 |
| 노련한 정치가 | 갈·영 | 개인 이득일 때만 분투, 겸손한 척 포장·조종 | **가장 위험** |
| 이상적인 팀 플레이어 | 겸·갈·영 | 세 덕목을 충분히 갖춤 ("이상적"≠"완벽") | **목표** |

### 다섯 함정 (1→5, 아래에서 위로 쌓임) + 덕목 연결

| 번호 | 함정 |
|------|------|
| 1 | 신뢰의 결여 (모든 함정의 뿌리) |
| 2 | 갈등에 대한 두려움 |
| 3 | 헌신의 결여 |
| 4 | 책임감의 회피 |
| 5 | 결과에 대한 무관심 |

- **겸손 → 신뢰의 토대** (1번 함정 차단, 취약성에 기반한 신뢰)
- **영리함 → 건전한 갈등** (2번 함정 차단, 감정 상하지 않는 솔직한 토론)
- **갈망 → 헌신·책임·결과** (3·4·5번 함정 차단)

## 7. 모듈 상세 — 연습 퀴즈 탭

**탭 ID** `tab-practice`

### 구성 요소

| 영역 | 내용 |
|------|------|
| 문항 카드 캐러셀 | 3문항 가로 스크롤, dot 인디케이터 |
| 선택지 | 4개 — 탭 시 즉시 정오답 표시 (초록/빨강) |
| 해설 | 정답 선택 후 펼침 표시 |
| 다음 문항 버튼 | 해설 아래 "다음 문항 →" |
| 마지막 카드 | "게이트 도전하기 →" + "↺ 다시 풀기" |

### 동작 원칙

- 한 번 선택하면 재선택 불가 (선택지 클릭 이벤트 제거)
- 정답은 초록(`#22c55e`) 테두리 + 배경, 오답은 빨강(`#ef4444`)
- "↺ 다시 풀기" → 전체 초기화 후 첫 카드로 이동

## 8. 모듈 상세 — 게이트 탭

**탭 ID** `tab-gate`

### 구성 요소

| 영역 | 내용 |
|------|------|
| 인트로 카드 | 채점 방식 안내 + 규칙 3개 + "첫 문항 풀기 →" |
| 문항 카드 (×5) | 번호 + "선택 전/✓ 선택됨" 배지 + 선택지 4개 |
| 다음 문항 버튼 | "다음 문항 →" — 해당 문항 선택 전 비활성, 선택 후 활성 |
| 하단 고정 버튼 | border-top 구분선 + "게이트 채점하기" |

### 버튼 활성화 규칙

| 버튼 | 비활성 조건 | 활성 조건 |
|------|-------------|-----------|
| 문항별 "다음 문항 →" | 해당 문항 미선택 (버튼은 항상 표시, disabled) | 해당 문항 선택 시 즉시 활성 |
| "게이트 채점하기" | 5문항 중 하나라도 미선택 (표시는 하되 disabled) | 5문항 전체 선택 시 활성 |

### 게이트 결과 바텀시트

| 상태 | 표시 내용 |
|------|-----------|
| **통과** (5/5) | 점수 + "모든 개념을 이해했습니다!" + 녹색 메시지(모듈 완료 처리) + **폭죽(컨페티) 축하 효과** + "다음 모듈로 →" (마지막 모듈은 "나의 여정 보기 →") |
| **미통과** | 점수 + 멘토 메시지(재도전 횟수별, 아래) + 틀린 문항 목록 (정답 표시) + "다시 도전하기" |
| 공통 | "학습 내용 다시 보기" (학습 탭으로 이동) |

### 미통과 멘토 메시지 (재도전 횟수별)

| 실패 횟수 | 메인 메시지 | 부가 안내 |
|-----------|-------------|-----------|
| 1회 | "괜찮습니다. 한번 더 공부하고 다시 도전해봐요." | (3회 실패 시 멘토에게 도움 메시지가 자동 발송될 예정입니다) |
| 2회 | "학습에 어려움이 있으신가요? 자료를 한번 더 읽어보고 재도전 해봐요." | (3회 실패 시 멘토에게 도움 메시지가 자동 발송될 예정입니다) |
| 3회 | "멘토와 함께 이 난관을 헤쳐나가 보아요. 멘토에게 도움의 메시지가 자동 발송됩니다." | — |

- 자동 발송 채널: **이메일**

## 9. 자기진단 화면

**화면 ID** `s-diag`

### 구성 요소

| 영역 | 내용 |
|------|------|
| 안내 배너 | 긍정 프레이밍 안내 + "정답이 없으니 솔직하게 응답해주세요" |
| 진행 표시 | `0 / 18 완료` |
| 설문 항목 | 공식 렌시오니 **자기참조형("나는 ~")** 문항 — 덕목별 6문항 × 3덕목 = 18문항 |
| 척도 버튼 | 3단계 — 언제나(3점) · 때때로(2점) · 거의 아님(1점) |
| 결과 화면 | 덕목별 점수 바 차트 + 개인 개발 초점 카드 |

> v1.2 변경: 기존 일반 서술형 문항을 공식 자기참조형 "나는 ~" 문장으로 전면 교체. 척도 최저 표기 "거의 안 함" → **"거의 아님"**.

### 설문 항목 목록 (자기참조형)

**겸손 (6문항)**

1. 나는 망설임 없이 칭찬하고 감사해한다.
2. 나는 내 실수를 순순히 인정한다.
3. 나는 내 약점을 진심으로 인정한다.
4. 나는 팀의 성취를 위해서라면 내 몫을 기꺼이 나눈다.
5. 나는 팀의 성공을 위해서라면 힘든 일을 기꺼이 맡는다.
6. 나는 정중하게 사과하고 품위 있게 사과를 받아들인다.

**갈망 (6문항)**

1. 나는 내 업무에서 요구되는 것 이상을 추구한다.
2. 나는 맡은 바 책임 외에 내가 기여할 수 있는 기회를 찾는다.
3. 나는 팀의 성공에 책임감을 느낀다.
4. 나는 근무시간 외에도 기꺼이 일하려 하고, 일에 관해 고민한다.
5. 나는 필요하다면 기꺼이 따분한 일을 맡으려 한다.
6. 나는 팀의 업무에 열정을 가지고 있다.

**영리함 (6문항)**

1. 나는 회의나 대화 시 동료들의 감정을 잘 이해한다.
2. 나는 팀 동료들에게 공감하는 모습을 나타낸다.
3. 나는 팀 동료들의 삶에 관심을 보인다.
4. 나는 경청을 잘한다.
5. 나는 나의 행동이 팀 동료들에게 어떤 영향을 미치는지 잘 안다.
6. 나는 나의 행동과 스타일을 대화나 관계의 성격에 맞게 고친다.

### 채점 방식 / 결과 해석

- 덕목별 점수 범위 **6~18점** (6문항 × 1~3점). 막대 차트는 6~18 범위로 표시
- 해석 기준
  - **16~18 강점** ("강점이에요")
  - **13~15 점검 필요** ("점검이 필요할 수 있어요")
  - **6~12 개발 영역** ("더 키울 영역이에요")
- 가장 낮은 덕목 1개 = **개인 개발 초점**으로 자동 지정
- 결과는 점수를 매기는 평가가 아니라 **스스로를 비춰보는 성찰 도구**임을 명시

### 결과 화면 구성

| 요소 | 내용 |
|------|------|
| 섹션 타이틀 | "나의 팀플레이어 현황" + "지금 이 순간의 스냅샷입니다" |
| 점수 바 차트 | 덕목별 점수(6~18) + 6~18 범위 막대 + 해석 라벨 1줄 |
| 개인 개발 초점 카드 | 가장 낮은 덕목 1개 강조 + 권장 모듈 범위 + "성찰 도구" 안내 문구 |
| CTA | "모듈 학습 시작하기 →" / "↺ 다시 진단하기" |

## 12. UX 카피 기준

### 원칙 5가지

| 원칙 | 규칙 | 예시 |
|------|------|------|
| 동사로 시작 | CTA는 반드시 행동 동사 | 풀기 / 시작하기 / 채점하기 |
| 목적어 명시 | "하기" 앞에 대상 명시 | ~~시작하기~~ → **게이트 채점하기** |
| 완료 계열 통일 | 진행 상태 표시어 | `N / N 완료` · `선택 전` · `✓ 선택됨` |
| 긍정 프레이밍 | "~아닙니다"로 시작 금지 | ~~인성 점수가 아닙니다~~ → **현황 확인 도구입니다** |
| 화살표(→) 규칙 | 화면 이동 시만 사용 | 결과 보기(→ 없음) / 다음 모듈로 →(있음) |

### 주요 CTA 확정 목록

| 위치 | 확정 문구 |
|------|-----------|
| 학습 탭 전체 완료 | 연습 퀴즈 풀어보기 → |
| 학습 탭 부분 완료 | 연습 퀴즈 시작하기 → |
| 게이트 인트로 | 첫 문항 풀기 → |
| 게이트 하단 | 게이트 채점하기 |
| 게이트 문항 간 | 다음 문항 → |
| 재인증 완료 후 | 여정으로 돌아가기 |
| 진단 결과 | 모듈 학습 시작하기 → |
