## 1. 프로젝트 개요

### 배경

케이옥션 IT팀 AI-native 전환 과정에서 조직 역량 내재화 목적으로 Patrick Lencioni의 *The Ideal Team Player* 기반 팀플레이어 학습 시스템을 구축한다.

### 학습 대상 — 3가지 덕목

| 덕목 | 영문 | 핵심 정의 | 모듈 |
|------|------|-----------|------|
| 겸손 | Humility | 팀의 성공을 개인의 성공보다 중요하게 여기는 태도 | M1 · M2 · M3 |
| 갈망 | Hunger | 외부 동기 없이 스스로 더 잘하려는 내적 의지 | M4 · M5 |
| 영리함 | Smarts | IQ가 아닌 사람의 감정과 상황을 읽는 능력 | M6 · M7 |

### 학습 여정 개요

```
자기진단 → 모듈 학습 (M1~M7) → 게이트 인증 → 착근기 (Day+7 / Day+21)
```

- 인증은 완성이 아닌 출발선
- 반복은 시스템에, 판단과 코칭은 사람에게

### 플랫폼

모바일 웹 앱 · 430px 기준 · 단일 HTML 파일 (prototype.html)

## 2. 화면 구조 및 플로우

### 화면 목록

| ID | 화면명 | 진입 방법 |
|----|--------|-----------|
| `s-home` | 홈 | 하단 네비 "홈" |
| `s-modules` | 모듈 목록 | 하단 네비 "모듈" |
| `s-module` | 모듈 상세 | 모듈 카드 탭 |
| `s-diag` | 자기진단 | 하단 네비 "진단" |
| `s-timeline` | 나의 여정 | 하단 네비 "여정" |

### 주요 사용자 플로우

```
[앱 첫 진입]
    │
    ▼
[홈 화면] ──────────────────────────────────────────┐
    │                                               │
    ▼                                               │
[자기진단] → 18문항 응답 → 결과 확인 → [모듈 학습 시작하기]
                                            │
                                            ▼
                                    [모듈 목록]
                                            │
                                            ▼
                                    [모듈 상세]
                                     ├─ 학습 탭 (카드뉴스 → 체크리스트)
                                     ├─ 연습 퀴즈 탭 (3문항 캐러셀)
                                     └─ 게이트 탭 (5문항 → 채점 → 바텀시트)
                                            │ 게이트 통과
                                            ▼
                                    [나의 여정] 타임라인 기록
                                            │ Day+7 활성화
                                            ▼
                                    [재인증 바텀시트] × 2회 (Day+7, Day+21)
```

### 화면 전환 방식

| 전환 | 방법 |
|------|------|
| 탭 간 이동 | 즉시 전환 (애니메이션 없음) |
| 모듈 목록 → 상세 | 오른쪽에서 슬라이드인 (280ms) |
| 모듈 상세 → 목록 | 오른쪽으로 슬라이드아웃 (250ms) |
| 게이트 결과, 재인증 | 바텀시트 슬라이드업 (380ms) |

## 3. 홈 화면

**화면 ID** `s-home`

### 구성 요소

| 영역 | 내용 | 비고 |
|------|------|------|
| 인사 영역 | 사용자 이름 + 부제 텍스트 | 상단 고정 |
| 벤 다이어그램 | 3덕목 블롭 애니메이션 | CSS border-radius keyframe, 겹침 영역 = 이상적 팀플레이어 |
| 진행 현황 카드 | 덕목별 완료 모듈 수 | 겸손 0/3 · 갈망 0/2 · 영리함 0/2 |
| 빠른 진입 CTA | 현재 진행 모듈 바로가기 | 상태에 따라 동적 표시 |

### 벤 다이어그램 명세

- 세 블롭이 유기적으로 형태 변화하며 서로 겹침
- 각 블롭: 덕목 색상 `rgba()` 24% 투명도 + 레이블 텍스트
- 겹침 중앙: "팀플레이어" 텍스트

## 5. 모듈 상세 — 학습 탭

**화면 ID** `s-module` / 탭 ID `tab-read`

### 구성 요소

| 영역 | 내용 |
|------|------|
| 헤더 | 뒤로가기 + 모듈 번호/제목 (sticky) |
| 모듈 히어로 | 제목 + 덕목 배지 + 설명 |
| 목표 확인 블록 | "이 모듈을 마치면" + 학습 목표 2~3개 (덕목 색상 체크 아이콘) |
| 탭 네비게이션 | 📖 학습 / ✏️ 연습 퀴즈 / 🔒 게이트 퀴즈 (sticky, top: 57px) |
| 카드뉴스 캐러셀 | 가로 스크롤 snap, dot 인디케이터 |
| 체크리스트 카드 | 핵심 내용 3항목 탭 → 완료 카운터 + CTA |

### 카드뉴스 슬라이드 구성 (M1 기준)

| 슬라이드 | 제목 | 내용 유형 |
|----------|------|-----------|
| 1 | 개념 소개 | 정의 + 본문 텍스트 |
| 2 | 핵심 내용 | 두 가지 위험한 유형 비교 |
| 3 | 체크리스트 | "지금 나의 겸손 체크" — 탭 체크 3항목 |

### 체크리스트 CTA 상태

| 상태 | 표시 요소 |
|------|-----------|
| 0개 완료 | 카운터만 표시 (CTA 숨김) |
| 1~2개 완료 | "연습 퀴즈 시작하기 →" (ghost 버튼) |
| 3개 모두 완료 | "연습 퀴즈 풀어보기 →" (채색 버튼) |

## 6. 모듈 상세 — 연습 퀴즈 탭

**탭 ID** `tab-practice`

### 구성 요소

| 영역 | 내용 |
|------|------|
| 문항 카드 캐러셀 | 3문항 가로 스크롤, dot 인디케이터 |
| 선택지 | 4개 — 탭 시 즉시 정오답 표시 (초록/빨강) |
| 해설 | 정답 선택 후 펼침 표시 |
| 다음 문항 버튼 | 해설 아래 "다음 문항 →" |
| 마지막 카드 | "게이트 도전하기 →" + "↺ 다시 풀기" |

### 동작 원칙

- 한 번 선택하면 재선택 불가 (선택지 클릭 이벤트 제거)
- 정답은 초록(`#22c55e`) 테두리 + 배경, 오답은 빨강(`#ef4444`)
- "↺ 다시 풀기" → 전체 초기화 후 첫 카드로 이동

## 8. 자기진단 화면

**화면 ID** `s-diag`

### 구성 요소

| 영역 | 내용 |
|------|------|
| 안내 배너 | 긍정 프레이밍 안내 + "정답이 없으니 솔직하게 응답해주세요" |
| 진행 표시 | `0 / 18 완료` |
| 설문 항목 | 덕목별 6문항 × 3덕목 = 18문항 |
| 척도 버튼 | 언제나(3점) · 때때로(2점) · 거의 안 함(1점) |
| 결과 화면 | 덕목별 점수 바 차트 + 집중 포인트 카드 |

### 설문 항목 목록

**겸손 (6문항)**

1. 팀 성과를 이야기할 때 "우리"라는 표현을 자연스럽게 사용한다
2. 아이디어가 채택되지 않아도 팀 결정을 열심히 실행한다
3. 동료의 성공을 진심으로 기뻐한다
4. 칭찬을 받으면 팀이나 다른 사람과 공을 나눈다
5. 실수를 인정하고 빠르게 사과한다
6. 나보다 잘하는 사람에게 배우는 것을 즐긴다

**갈망 (6문항)**

1. 일을 더 잘하고 싶다는 의욕이 자연스럽게 생긴다
2. 업무 목표를 달성했을 때 다음 목표를 스스로 찾는다
3. 어렵거나 도전적인 과제가 주어지면 의욕이 오른다
4. 추가 책임이나 역할을 기꺼이 받아들인다
5. 퇴근 후에도 업무 개선 아이디어가 떠오른다
6. 현재 방식보다 더 나은 방법을 찾아 제안한다

**영리함 (6문항)**

1. 상대방의 감정 상태를 빠르게 파악한다
2. 대화 중 분위기가 어색해지면 자연스럽게 개입한다
3. 내 말이나 행동이 상대에게 어떻게 느껴질지 미리 생각한다
4. 실수한 동료가 당황하지 않도록 자연스럽게 돕는다
5. 자신의 감정이 대화에 미치는 영향을 인식한다
6. 갈등 상황에서 양측 입장 모두를 이해하려 한다

### 채점 방식

- 덕목별 최고 18점 (6문항 × 3점)
- 결과는 점수 바 차트로 시각화 (% 변환)
- 최저 점수 덕목 = 집중 포인트로 자동 지정
- 비판단적 피드백: 점수 낮음 = "개발할 여지가 큽니다" 표현

### 결과 화면 구성

| 요소 | 내용 |
|------|------|
| 섹션 타이틀 | "나의 팀플레이어 현황" + "지금 이 순간의 스냅샷입니다" |
| 점수 바 차트 | 덕목별 점수/18 + 퍼센트 바 + 비판단 피드백 1줄 |
| 집중 포인트 카드 | 최저 덕목 강조 + 권장 모듈 범위 |
| CTA | "모듈 학습 시작하기 →" / "↺ 다시 진단하기" |

| 착근기 대기 | 점선 테두리 dot + opacity 0.35 |
| 착근기 활성 | opacity 1.0 + "재인증 시작하기 →" 버튼 노출 |

### 타임라인 구조 (순서)

```
● 여정 시작 (날짜)
├─ M1 겸손의 의미와 중요성
├─ M2 겸손한 사람의 특징
├─ M3 겸손 실천하기
├─ 🎯 겸손 완성 (M1~M3 게이트 통과)
├─ M4 내적 갈망의 힘
├─ M5 자기동기부여
├─ 🎯 갈망 완성 (M4~M5 게이트 통과)
├─ M6 대인관계 영리함
├─ M7 팀 내 소통 영리함
├─ 🎯 영리함 완성 (M6~M7 게이트 통과)
├─ 🏅 캡스톤 인증
├─ Day+7 재인증 (1차 확인)
└─ Day+21 재인증 (2차 습관 안착)
```

## 11. UX 카피 기준

### 원칙 5가지

| 원칙 | 규칙 | 예시 |
|------|------|------|
| 동사로 시작 | CTA는 반드시 행동 동사 | 풀기 / 시작하기 / 채점하기 |
| 목적어 명시 | "하기" 앞에 대상 명시 | ~~시작하기~~ → **게이트 채점하기** |
| 완료 계열 통일 | 진행 상태 표시어 | `N / N 완료` · `선택 전` · `✓ 선택됨` |
| 긍정 프레이밍 | "~아닙니다"로 시작 금지 | ~~인성 점수가 아닙니다~~ → **현황 확인 도구입니다** |
| 화살표(→) 규칙 | 화면 이동 시만 사용 | 결과 보기(→ 없음) / 다음 모듈로 →(있음) |

### 주요 CTA 확정 목록

| 위치 | 확정 문구 |
|------|-----------|
| 학습 탭 전체 완료 | 연습 퀴즈 풀어보기 → |
| 학습 탭 부분 완료 | 연습 퀴즈 시작하기 → |
| 게이트 인트로 | 첫 문항 풀기 → |
| 게이트 하단 | 게이트 채점하기 |
| 게이트 문항 간 | 다음 문항 → |
| 재인증 완료 후 | 여정으로 돌아가기 |
| 진단 결과 | 모듈 학습 시작하기 → |
